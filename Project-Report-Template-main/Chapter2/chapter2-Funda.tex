\chapter{Fundamentals and Literature Survey}

This chapter explains the concepts used while designing VisuaLearn AI and reviews earlier work related to tutoring systems, adaptive learning, visualization, simulation, and AI-assisted education. The aim is to show why the project was designed as more than a normal chatbot. VisuaLearn AI combines conversation, generated learning resources, learner context, visual support, and stored revision material in one application.

\section[Fundamentals]{\textbf{Fundamentals}}

\subsection[Intelligent Tutoring Systems]{\textbf{Intelligent Tutoring Systems}}

An Intelligent Tutoring System (ITS) is a learning system that gives instruction and feedback based on the learner's current state. A typical ITS contains four parts: a domain model, a learner model, a tutoring model, and an interface model. The domain model stores subject knowledge. The learner model estimates what the student knows. The tutoring model decides the next instructional action. The interface model presents that action to the learner.

VisuaLearn AI follows this idea in a web application setting. The subject content is not fixed to one domain. It is generated for the topic asked by the learner. The learner model is built from conversation history, quiz responses, resource usage, engagement signals, and profile information. The tutoring model selects response formats such as concise explanation, guided steps, quiz mode, visual learning, Socratic questioning, or remediation.

\subsection[Adaptive Learning]{\textbf{Adaptive Learning}}

Adaptive learning changes instruction according to the learner's needs. A learner who answers correctly may be ready for a harder example. A learner who repeatedly struggles may need a slower explanation or a simpler example. A learner revising for a test may need questions and flashcards instead of a long explanation.

In this project, adaptation is applied at the response level. The system does not only ask which topic should be taught next. It also asks how the current response should be structured. The adaptive layer uses cognitive state, engagement level, topic status, performance trend, confidence, topic difficulty, previous failures, and response time to choose the teaching mode.

\subsection[Educational Visualization and Simulation]{\textbf{Educational Visualization and Simulation}}

Educational visualization uses diagrams, charts, animations, and interactive views to represent ideas that are difficult to follow in text. It is useful when a topic involves structure, movement, sequence, or relationships. In computer science, visualization can help students study algorithms, data structures, recursion, memory use, and state transitions.

Simulation-based learning is useful when a concept can be represented as ordered states. Sorting can be shown through comparisons and swaps. Graph traversal can be shown through visited nodes and queue changes. CPU scheduling can be shown through a timeline of process execution. Such representations help students inspect intermediate steps instead of only reading the final answer.

\subsection[Structured Revision]{\textbf{Structured Revision}}

Understanding a topic once is not enough for retention. Students need revision material that can be used later. Flashcards support recall. Quizzes support self-assessment. Mind maps help organize related ideas. Examples help learners apply the same concept in a new situation. VisuaLearn AI generates these resources from the learning session so that the learner can move from explanation to revision without creating all material manually.

\section[Literature Survey]{\textbf{Literature Survey}}

Bloom \cite{Bloom1984} reported the value of individualized tutoring through the well-known two-sigma problem. The work is important because it shows why personalization matters in education. Its practical limitation is scale. A human tutor cannot be assigned continuously to every learner, which creates a need for software-supported tutoring.

VanLehn \cite{VanLehn2011} reviewed the effectiveness of human tutoring, intelligent tutoring systems, and related tutoring systems. The review shows that the quality of interaction affects tutoring performance. This is relevant to VisuaLearn AI because the system is designed to choose the form of each response, not just the topic of the next lesson.

Ma et al. \cite{Ma2014} presented a meta-analysis of intelligent tutoring systems and learning outcomes. Their findings support learner modeling and timely feedback as useful educational mechanisms. At the same time, many ITS implementations are built for fixed subject domains. They do not always support open-ended questions, generated visual material, and multimodal revision resources.

Mayer \cite{Mayer2009} and Sweller \cite{Sweller1988} provide the theoretical base for combining verbal and visual explanation while controlling mental load. These theories are useful for the design of visual explanations, simulations, and stepwise content in VisuaLearn AI. They do not, however, describe how to implement a complete AI learning platform with authentication, storage, routing, validation, and feedback.

Kasneci et al. \cite{Kasneci2023} discuss the educational opportunities and risks of large language models. Such models can generate explanations and respond to natural-language questions, but they may also produce incorrect or unsupported content. This influenced the project design, especially the use of backend orchestration, validation, feedback collection, and persistent learning history.

Chen et al. \cite{Chen2020} review artificial intelligence in education and highlight personalization, feedback, and responsible deployment as important concerns. Their work supports the decision to treat VisuaLearn AI as an application with user accounts, stored sessions, adaptive routing, and safe rendering rather than as a standalone answer generator.

\section[Research Gap]{\textbf{Research Gap}}

The reviewed work shows that several useful learning features already exist, but they are often implemented separately. Some systems focus on tutoring dialogue. Some focus on quizzes. Some focus on visualization or simulation. Others focus on learner modeling or course delivery. A student who needs all of these supports may still have to move between different tools.

The main gaps identified from the review are:
\begin{itemize}[leftmargin=1.2cm]
\item Many systems provide explanations but do not generate a complete set of learning resources from the same session.
\item Visualization and simulation are often separate from the tutoring workflow.
\item Revision content is usually prepared manually by the learner.
\item Learner context is not always used to decide the form of each response.
\item Generated visual or interactive content requires validation before it is shown to users.
\item Many LLM-based tools do not preserve learning history, feedback, and generated artifacts in an organized way.
\end{itemize}

\section[Proposed Solution]{\textbf{Proposed Solution}}

VisuaLearn AI addresses these gaps by combining conversational tutoring, adaptive response selection, document-based learning, visual resources, simulations, quizzes, flashcards, feedback, and persistent history in one web application. The system is designed so that a learner can ask a question, receive an explanation, open supporting learning resources, revise the topic, and return to the saved session later.

The project also treats generated learning output as a managed resource. Responses, visual widgets, simulations, quizzes, flashcards, and notes are linked to the learner session. Authentication, route protection, validation, fallback handling, and logging are included to make the generated resources more reliable within the application workflow.

\section[Summary]{\textbf{Summary}}

The literature shows strong support for personalization, visual explanation, adaptive feedback, and structured revision. It also shows that these features are often studied or implemented separately. VisuaLearn AI is positioned as an integrated project that brings these ideas into one working system and prepares it for usability and learning-support evaluation.
